# Roadmap

## 단계 요약

| 단계 | 상태 | 핵심 | 기간 목표 |
|---|---|---|---|
| v0 | 진행 중 | LLM-only 직판정 | 1주 |
| v1 | 미시작 | 룰 1차 필터 + LLM 2차 | 데이터 늘면 |
| v2 | 후보 | ML 분류기 도입 검토 | 라벨 200건+ 모이면 |

## v0 — LLM-only

목표: 폴더 watch → 엑셀 로드 → 로컬 LLM이 룰+상식 추론으로 이상 행 판정 → 메일 발송.

### 채택 이유

- 데이터가 데모 수준이라 ML 학습 자체가 불가 (최소 수천 행 필요)
- 라벨 유무 미확인 → 지도학습 시작 못 함
- 로컬 LLM 인프라가 정책상 어차피 필요 → LLM-only로 시작이 자연스러움

### 완료 정의

- 합성 엑셀 1건 떨어뜨리면 메일이 정상 발송됨
- 룰은 프롬프트로 표현됨 (코드 변경 없이 갱신 가능)
- 실 데이터 0% 포함된 합성 데이터로 동작 검증

## v1 — 룰 + LLM

LLM 호출 빈도/비용을 줄이기 위한 1차 룰 필터 추가. 명백한 케이스(예: 새벽 출입, 권한등급 미스매치)는 결정론적 룰로 1차 처리, 애매한 것만 LLM 호출.

### 진입 조건

- v0 운영 중 LLM 호출량이 의미 있게 증가 (구체 임계는 운영 중 결정)
- 명백한 케이스가 LLM 추론 없이도 결정 가능함을 확인

## v2 — ML 분류기 검토

지도학습 분류기 도입 여부 결정.

### 진입 조건 (모두 만족 시에만)

- 라벨 데이터 200건 이상 확보
- LLM v0/v1 baseline 정확도 측정 완료 (PR-AUC 등)
- ML 분류기가 LLM baseline을 의미 있게 능가하는지 작은 실험으로 검증

후보 설계는 [explorations/ml-pipeline.md](explorations/ml-pipeline.md). **진입 조건 미충족 시 도입하지 않음.** 잘못 도입하면 데이터 부족 / 라벨 부족으로 ML 정확도가 LLM보다 떨어질 위험.

## 단계 게이트 원칙

다음 단계로 넘어가기 전 반드시:
1. 현 단계 운영 측정값 보유 (정확도, 호출량, 비용)
2. 다음 단계 가설이 측정값으로 정당화되는지 확인
3. 정당화 안 되면 단계 보류, 현 단계 개선에 집중
